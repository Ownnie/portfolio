## Desafíos técnicos resueltos

### Motor de elegibilidad médica (MEC)

Implementación del algoritmo de criterios médicos de elegibilidad basado en guías de OMS, ACOG y MinSalud:

- **Categorización 1–4**: Sistema de clasificación automático que evalúa la seguridad de cada método según condiciones específicas
- **Reglas dinámicas complejas**: 
  - Posparto/lactancia con intervalos (menos de 21, 21–42, más de 42 días) y riesgo de TEV
  - Tabaquismo ≥35 años con umbral de 15 cigarrillos/día (categoría 3 vs 4 para AHC)
  - Migraña con/sin aura, HTA grave, TVP/EP, trombofilias, diabetes vascular
  - Postaborto séptico, sangrado abundante, y más de 40 condiciones evaluadas
- **Flujo condicional inteligente**: Preguntas que se adaptan según respuestas previas (edad, embarazo, nuliparidad, etc.)
- **Arquitectura modular**: Separación clara entre `engine.ts` (lógica de evaluación), `mappings.ts` (reglas clínicas) y `questions.ts` (cuestionario)

### Privacidad y tratamiento de datos local-only

Diseño centrado en privacidad sin infraestructura de backend:

- **LocalStorage exclusivo**: Todas las respuestas del test se almacenan únicamente en el navegador del usuario
- **Consentimiento robusto**: Modal con dos checkboxes (aviso médico + verificación ≥14 años) persistido localmente
- **Sin tracking**: Cero identificadores personales, cookies de terceros o envío de datos a servidores
- **Página de privacidad**: Documentación clara de tratamiento de datos y disclaimers legales

### Arquitectura de contenido JSON-driven

Sistema escalable y mantenible para gestionar información médica:

- **Separación de datos y UI**: Fichas de métodos en JSON estructurado (`/content/methods/*.json`)
- **Versionamiento facilitado**: Cambios en contenido médico sin tocar código
- **Preparado para i18n**: Estructura lista para internacionalización
- **Imágenes dinámicas**: Sistema de mapping para asociar imágenes por método (`METHOD_IMAGES`)

### Correcciones técnicas Next.js 16

Migración y optimización para App Router moderno:

- **Async params**: Corrección de acceso a `params` como Promise en rutas dinámicas
- **Viewport export**: Separación de configuración de viewport según nuevas APIs
- **Metadata dinámica**: `generateMetadata` async para SEO por método
- **Node >=20.9**: Actualización de runtime y dependencias

## Funcionalidades principales

### Fichas de métodos (`/metodos`)

- **Listado con búsqueda y filtros**: Hormonal, No hormonal, LARC, Emergencia
- **12 métodos cubiertos**: DIU (Cu/LNG), Implante (3/5 años), Inyectables (mensual/trimestral), Píldoras (combinadas/solo progestina), Parche, Anillo, Barrera, Píldora de emergencia, Métodos naturales
- **Tarjetas informativas**: Badges, descripción breve, imagen representativa
- **Fichas detalladas por método**: Datos rápidos, eficacia (típica/perfecta), ventajas, consideraciones, CTA al test

### Test interactivo MEC (`/test`)

- **Cuestionario adaptativo**: 40+ preguntas condicionales según perfil de salud
- **Evaluación en tiempo real**: Clasificación instantánea de todos los métodos (1–4)
- **Resultados personalizados**: Recomendaciones basadas en evidencia médica
- **Persistencia local**: Estado guardado en navegador para retomar después

### Comparador de métodos (`/comparar`)

- **Selección múltiple**: Hasta 3 métodos simultáneos
- **Tabla responsive**: Comparación lado a lado de características clave
- **Criterios comparados**: Tipo/hormonas, duración, protección ITS, eficacia, uso como emergencia, ventajas y consideraciones
- **UX optimizada**: Pills seleccionables, encabezados destacados, badges informativos

### Home y navegación

- **Hero impactante**: CTA clara hacia el test
- **Secciones educativas**: Objetivo, necesidad, PICO de pregunta clínica
- **Header/Footer consistentes**: Navegación principal, aviso de uso educativo y emergencias
- **Diseño coherente**: Paleta rosa + blanco, gradientes, "underbar", tarjetas hover

## Mejoras de UX y contenido

- **Copy sin marcas comerciales**: "Píldora de emergencia" en lugar de nombres comerciales
- **DIU de cobre dual**: Marcado como anticonceptivo regular Y método de emergencia
- **Métodos de barrera visibles**: Con nota clara sobre protección contra ITS
- **Métodos naturales incluidos**: Cobertura completa del espectro anticonceptivo
- **Mobile-first**: Controles simples, tipografías legibles, espaciado generoso
- **Accesibilidad**: Badges claros, contraste adecuado, navegación intuitiva

## Impacto y resultados

- **Educación basada en evidencia**: Información actualizada según guías internacionales (OMS/ACOG/MinSalud)
- **Autonomía informada**: Usuarias pueden explorar opciones según su perfil de salud específico
- **Privacidad garantizada**: Cero riesgos de filtración de datos sensibles de salud
- **Escalabilidad**: Arquitectura preparada para agregar métodos, idiomas y actualizaciones médicas
- **Referencias intactas**: Todas las cifras de eficacia y recomendaciones mantienen respaldo científico original

## Stack técnico detallado

- **Framework**: Next.js 16 (App Router) + TypeScript estricto
- **Estilos**: Tailwind CSS 3.5+ con configuración personalizada
- **Persistencia**: LocalStorage API para estado del test y consentimiento
- **Contenido**: JSON estructurado con validación de tipos
- **Deploy**: Vercel (edge functions, CDN global)
- **Tooling**: ESLint + Prettier, Git flow estándar
